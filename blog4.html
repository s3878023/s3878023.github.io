BRANCHING

the image link

Branch is what is used to branch and record the flow of history. Branched branches will not affect another branch, so you can perform many changes simultaneously in the same repository.

Furthermore, a branched branch can be modified to merge into one branch by merging with another branch.

There are a few simple manipulations in branching: Create the branch on your local PC and switch into working on this new branch : * git checkout -b [name_of_your_new_branch]

Push the branch on github : * git push origin [name_of_your_new_branch]
Add a new remote for your branch : * git remote add [name_of_your_remote] [name_of_your_new_branch]

Push changes from your PC (comit) into your branch : * git push [name_of_your_new_remote] [url]

Delete a branch on the local PC: * git branch -d [name_of_your_new_branch]

Force to delete on your local PC : * git branch -D [name_of_your_new_branch]

Delete the branch on github : * git push origin :[name_of_your_new_branch]

Update your branch when the original one when the official repository has been changed * git fetch [name_of_your_remote]

You can see all the branches created by using : * git branch -a
